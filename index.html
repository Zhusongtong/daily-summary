大约一年前我构建 menugen 时，我注意到迄今为止最难的部分并非代码本身，而是需要像组装宜家家具一样把大量的服务拼凑起来才能让项目真正运行——包括各种 DevOps 相关的服务、支付、认证、数据库、安全、域名等……我真的期待有一天，我只需要对代理说：“构建 menugen”（指那篇帖子），它就能自动完成一切——从完整的前端页面到部署上线。代理需要浏览多个服务、阅读文档、获取所有 API 密钥、让所有功能运转、在开发环境调试并部署到生产环境。这才是真正困难的部分，而非代码本身。换句话说，更好的思路是：整个 DevOps 生命周期必须变成代码，再加上必要的 CLI/API 传感器/执行器，以代理原生的人机交互方式呈现，而且人类无需再访问网页、点击按钮或做类似的事情。说起来容易，如今也勉强在技术上可行，甚至有望实现，但它绝对需要从头重新设计、工作和思考。这是一个非常令人振奋的方向！
